# Workflow Discovery Questions

## Operational Overview

- What initiates the workflow?
- What is the final operational outcome?
- Which teams or people are involved?
- Which stages are manual?
- Which stages already involve AI assistance?

---

# Workflow Structure

- What are the current workflow stages?
- Which stages depend on approvals?
- Which stages frequently slow down?
- Which stages are difficult to trace later?
- Where does operational ambiguity appear?

---

# Decision Flow

- Where are operational decisions made?
- Who owns each decision?
- Which decisions are repeated frequently?
- Which decisions are undocumented?
- Which decisions create bottlenecks?

---

# AI Integration

- Where is AI currently used?
- What type of outputs does AI generate?
- Who validates AI outputs?
- What happens if AI output is incorrect?
- Which AI-assisted stages are fragile?

---

# Dependency Analysis

- Which stages depend heavily on one person?
- Which stages rely on undocumented knowledge?
- Which tools are operationally critical?
- Which failures create cascading problems?
- Which dependencies are currently hidden?

---

# Governance & Boundaries

- Where should AI outputs stop?
- Which stages require human validation?
- Where should escalation occur?
- Which actions should never be automated?
- Which operational boundaries are currently unclear?

---

# Maintainability

- Which workflows are becoming harder to maintain?
- Which systems are difficult to modify safely?
- Which processes rely on tribal knowledge?
- Which workflows become unstable as complexity increases?

---

# Objective

The purpose of workflow discovery is not only automation.

The purpose is operational clarity.
